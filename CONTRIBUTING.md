# Contributing to QTI 3.0 Scoring System

Thank you for your interest in contributing to the QTI 3.0 Scoring System!

## Getting Started

1. Fork the repository.
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/qti-scoring-system.git`
3. Install dependencies: `npm install`
4. Create a new branch: `git checkout -b feature/your-feature-name`

## Development Workflow

- Run `npm run dev` for local development.
- Write tests for new features.
- Ensure all tests pass: `npm run verify`
- Format your code: `npm run format`

## Pull Request Process

1. Ensure your code adheres to the project's coding standards.
2. Update the README if necessary.
3. Submit a pull request to the `main` branch.
4. Provide a clear description of the changes and the problem they solve.

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
